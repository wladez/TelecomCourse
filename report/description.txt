Платёжная система.
Должна быть аутентификация пользователей. Каждый пользователь входит по имени и паролю, у каждого пользователя есть свой кошелёк с уникальным номером. При регистрации пользователь получает N у.е.
 
Пользователи могут:
	смотреть список всех пользователей, чтобы узнать номер чьего-либо кошелька;
	переводить у.е. на другой кошелёк, знаю его номер

Сервер должен:
	хранить список пользователей и номера их кошельков;
	регистрировать новых пользователей;
	работать одновременно с несколькими пользователями;
	принудительно отключать пользователей (тут есть вопросы).
